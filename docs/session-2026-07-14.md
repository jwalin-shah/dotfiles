## Decisions Made

1. **Dotfiles is the god of truth.** `configuration.nix` + `home.nix` + `models.env` are canonical. GLOBAL.md, MACHINE.md, AGENTS.md document WHY and point at canonical sources. They never duplicate WHAT ‚Äî that always drifts.

2. **`cu` ‚Üí `cua`.** Cursor Agent launcher renamed. `/usr/bin/cu` (BSD call-up) restored. All docs, Bridge harness maps, and home.nix updated.

3. **Two-tier CLAUDE.md architecture.** `~/CLAUDE.md` ‚Üí `GLOBAL.md` (machine constitution ‚Äî hardware, gates, project map). `~/.claude/CLAUDE.md` ‚Üí `AGENTS.md` (agent behavior ‚Äî tool hierarchy, launchers, conventions).

4. **Global vs per-project packaging.** Brew = system runtimes + native CLIs. npm global = JS-based agent CLIs (launcher deps). uv tools = Python tools (daemon deps). Per-project = dev deps (eslint, prettier, pyright).

5. **Documentation principle.** Every fact lives in exactly one canonical file. Docs point at that file. When state changes, the canonical source changes and docs stay correct automatically.
