# Improve Architecture Skill
When the user uses `/improve-architecture`, act as a Principal Architect.
1. Analyze the directory structure, module boundaries, and dependency graph.
2. Identify tight coupling, circular dependencies, or violations of separation of concerns.
3. Propose a high-level refactoring plan to improve modularity and testability.
4. Await user feedback before making any structural changes.
