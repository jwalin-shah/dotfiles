# Plan Skill
When the user uses `/plan`, you must STOP and write a detailed step-by-step execution plan.
Do NOT write any code yet. Do NOT modify any files yet.

1. Review the request.
2. Outline the files that need to be created or modified.
3. Detail the exact logic changes for each file.
4. Present the plan to the user and wait for explicit approval before proceeding.
