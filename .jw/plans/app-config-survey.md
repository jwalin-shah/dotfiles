## Summary

Two apps (aerospace, karabiner) already follow the correct dotfiles-managed symlink pattern; Cursor correctly excludes its runtime-rewritten file. The one real gap is **ghostty** — its config is a plain uncommitted file, not symlinked, so it won't survive a fresh machine setup; fix is a one-line `home.file` entry mirroring the aerospace pattern, no auto-reload concern since Ghostty already watches its config live. Brave's Preferences JSON should explicitly stay unmanaged (same reasoning as Cursor's `cli-config.json` exclusion) — recommend a macOS configuration profile instead if policy-level Brave settings need to be durable. Raycast, LuLu, and MonitorControl store settings in binary plists or weren't found configured at all; low priority, and binary plists resist clean version control regardless.
